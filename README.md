# Database_HW - Базы данных
- В данном репозитории содержится информация о заданиях по работе с базами данных. Задания охватывают создание, наполнение и манипуляцию данными в базах данных.
---
### Используемая СУБД

- PostgreSQL **14+**
- Среда выполнения запросов: DBeaver

### Инструкция по запуску

1. Создать одну из баз данных
- db1_transport - транспортные средства
- db2_races - автомобильные гонки
- db3_hotels - бронирование отелей
- db4_organizations - структура организации
2. Открыть одноимённую папку
3. Запустить создание таблиц - файл create_tables.sql
4. Запустить скрипты наполнения данными - файл insert_tables.sql
5. Выполнить запросы по задачам - файл db_query.sql
---
### Цели и задачи
- Создание структуры базы данных для хранения информации.
- Наполнение базы данных тестовыми данными.
- Реализация SQL-запросов для работы с информацией.

### Общий обзор
В рамках проекта были реализованы аналитические запросы для четырех предметных областей:
1. Транспортные средства (db1_transport) — учет автомобилей, мотоциклов и велосипедов
2. Автомобильные гонки (db2_races) — анализ результатов гоночных соревнований
3. Бронирование отелей (db3_hotels) — анализ предпочтений клиентов и статистика бронирований
4. Структура организации (db4_organizations) — управление иерархией сотрудников, проектами и задачами

### Применённые технические решения
#### Используемые возможности PostgreSQL
- WITH ... AS для структурирования запросов
- WITH RECURSIVE для обхода иерархий
- Агрегатные функции: AVG(), COUNT(), SUM(), MIN(), MAX()
- Функции для работы со строками: STRING_AGG()
- Условная логика: CASE, COALESCE
#### Паттерны проектирования запросов
- Фильтрация через HAVING — для условий на агрегированные данные
- LEFT JOIN для сохранения NULL — чтобы не терять записи без связей
- DISTINCT при агрегации — для устранения дубликатов
- Вложенные CTE(Common Table Expressions) — для пошагового вычисления сложных метрик

### Примечания
Все решения оформлены в виде SQL-скриптов с комментариями для удобства проверки.
